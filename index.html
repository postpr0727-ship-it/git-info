GitHub 업로드 가이드

프로젝트 디렉토리 찾는 방법

터미널에서 (가장 빠른 방법):
  1. 프로젝트 폴더로 이동 (이미 있는 경우 생략)
  2. 터미널에서 다음 명령어 실행:
# Git Bash / Linux / Mac
pwd

# Windows 명령 프롬프트 (CMD)
cd

# Windows PowerShell
pwd 또는 Get-Location
Windows CMD 팁:
  • cd - 현재 경로 표시 (가장 간단)

출력된 경로를 복사하여 아래 입력창에 붙여넣기

프로젝트 디렉토리 경로 입력:
경로를 입력하면 아래 명령어에 자동으로 적용됩니다.
저장된 프로젝트 선택:
자주 쓰는 프로젝트 경로를 저장해 두고 선택해서 쓸 수 있습니다.
GitHub 저장소 URL 입력:
GitHub 저장소 URL을 입력하면 최초 설정 명령어에 자동으로 적용됩니다.

최초 설정 (한 번만 실행)

주의: 아래 명령어들은 프로젝트 최초 설정 시 한 번만 실행하면 됩니다.
특히 git push -u origin main 은 처음 한 번만 실행해서 GitHub와 브랜치를 연결해 줍니다.
# 프로젝트 디렉토리로 이동 cd D:\CURSOR\siteplan # Git 저장소 초기화 (한 번만) git init # GitHub 연결 (한 번만) git remote add origin https://github.com/postpr0727-ship-it/sitemap-plan.git # 기본 브랜치 이름 설정 (한 번만) git branch -M main # 최초 1회: GitHub와 main 브랜치 연결 + 첫 푸시 git push -u origin main

반복 작업 (일반적인 업로드)

# 프로젝트 디렉토리로 이동 cd D:\CURSOR\siteplan # 변경사항 스테이징 git add . # 커밋 (작업 내용을 요약해서 작성) git commit -m "작업 내용 요약" # GitHub에 푸시 git push

유용한 팁

Git 명령어 팁:
  • git status - 현재 상태 확인
  • git log - 커밋 히스토리 확인
  • git pull - 원격 저장소에서 최신 변경사항 가져오기

  • 커밋 메시지는 작업 내용을 명확하게 작성하세요
  • 최초 설정 후에는 반복 작업 명령어만 사용하면 됩니다
  • GitHub 저장소 URL은 프로젝트에 맞게 변경하세요
